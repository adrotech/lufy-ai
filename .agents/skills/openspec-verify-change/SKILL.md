# OpenSpec Verify Change

Use after implementation to compare code/docs/tests with proposal, specs, and tasks.

1. Check tasks, scenarios, acceptance criteria, and structural constraints.
2. Run proportional validation commands when available.
3. Report gaps as blocked or needs_revision.
4. Do not archive or deliver without required evidence.
