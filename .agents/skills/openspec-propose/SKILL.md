# OpenSpec Propose

Use for T1 or substantial T2 changes that need a proposal before implementation.

1. Define change ID, problem, goals, non-goals, acceptance criteria, and tasks.
2. Add spec deltas with scenarios using WHEN/THEN.
3. Preserve optional HTML overview outcome: generated, offered_pending, skipped_by_user, or not_available.
4. Do not implement runtime changes in the proposal step.
