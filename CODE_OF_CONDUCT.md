# Código de conducta

Este proyecto espera interacción profesional, respetuosa y enfocada en mejorar el software.

## Comportamientos esperados

- Dar feedback técnico concreto y accionable.
- Asumir buena intención mientras se corrigen problemas con claridad.
- Respetar el tiempo de mantenedores y contribuidores.
- Mantener discusiones públicas libres de información sensible.

## Comportamientos no aceptados

- Acoso, insultos o ataques personales.
- Publicación de secretos, credenciales o datos privados.
- Spam o presión para mergear cambios sin revisión suficiente.

Los mantenedores pueden moderar issues, PRs o discusiones para proteger la calidad y seguridad del proyecto.
