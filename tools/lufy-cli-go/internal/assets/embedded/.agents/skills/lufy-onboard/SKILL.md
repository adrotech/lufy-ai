# Lufy Onboard

Use when the user asks how to start using Lufy in a repo.

1. Inspect install state, `AGENTS.md`, and `.lufy/config/project.yaml`.
2. Explain current tool/methodology selection and detected surfaces.
3. Offer a safe T3 dry-run style demo before mutating files.
4. Keep delivery separate and explicitly authorized.
